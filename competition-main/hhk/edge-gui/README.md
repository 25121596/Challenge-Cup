# Edge GUI — 边端 LLM 推理桌面应用

## 1. 这是什么？

Edge GUI 是一个轻量级的桌面端大语言模型对话工具。它直接在你的电脑上运行量化后的模型（无需联网），也可连接到云端 API。专为内存受限（< 1.5 GB）的边缘设备设计。

**核心能力**：本地推理 + 云端卸载 + 自动回退 + 流式对话 + 会话管理。

---

## 2. 快速上手

### 2.1 启动程序

编译后，可执行文件位于：

```
build\bin\Release\edge-gui.exe     （Release 版本，推荐）
build\bin\Debug\edge-gui.exe       （Debug 版本）
```

```sh
# 直接启动
build\bin\Release\edge-gui.exe

# 或指定模型
build\bin\Release\edge-gui.exe -m path\to\your-model.gguf
```

### 2.2 放置模型文件

下载好的 `.gguf` 模型文件可以放在**任意位置**，支持两种加载方式：

| 方式 | 操作 | 适合场景 |
|------|------|---------|
| **打开** 按钮 | 手动选择文件 | 模型放在任意目录 |
| **模型** 按钮 | 自动扫描列表 | 模型放在 `models\` 目录下 |

**Models 按钮自动扫描**：程序启动时自动从 exe 位置推导项目根目录（如 `D:\llama.cpp\`），然后扫描 `D:\llama.cpp\models\` 下的所有 `.gguf` 文件。

也可以在 **设置 → 模型目录** 中修改扫描路径，点击"应用并刷新"生效。

### 2.3 开始对话

1. 点击 **打开** 选择一个 `.gguf` 文件，或点击 **模型** 从列表中选择
2. 如需要，在 **设置 → 内存预算** 中设置设备的可用内存
3. 在底部输入框打字，按 Enter 或点击 **发送**

### 3.1 顶部状态栏

| 显示 | 说明 |
|------|------|
| `[模型] 文件名` | 绿色 = 已加载；红色 = 未加载 |
| `[加载中 ...]` | 黄色 = 正在加载模型 |
| `预处理: 520 t/s` | 上轮推理的 Prompt 处理速度 |
| `生成: 45 t/s` | 上轮推理的 Token 生成速度 |

| `[云端]` | 蓝色 = 云端已连接；橙色 = 已回退本地 |

### 3.2 顶部按钮

| 按钮 | 功能 |
|------|------|
| **打开** | 弹出文件对话框，选择任意位置的 `.gguf` 模型 |
| **模型** | 打开模型浏览器，列出 `models\` 下所有模型 |
| **设置** | 生成参数 / 采样 / 模型目录 / 内存预算 / 系统提示词 |
| **云端** | 配置远程 API 端点 |
| **保存** | 保存当前对话 (JSON + KV 缓存快照) |
| **加载** | 恢复之前保存的对话 |

### 3.3 输入区域

- **正常状态**：文本框 + **发送** 按钮（灰色 = 无模型时禁用）
- **推理中**：文本框消失，显示红色 **停止** 按钮（点击立即中断生成）
- 快捷键：Enter 发送

## 4. 模型浏览器

点击 **模型** 按钮打开，自动扫描 `models\` 目录。

```
┌──────────────────────────────────────────────────────┐
│ 可用模型                                 [刷新]        │
│ 目录: D:\llama.cpp\models (1 个模型)                  │
├──────────────────────────────────────────────────────┤
│ 模型                            │ 大小   │ 适配 │     │
├──────────────────────────────────────────────────────┤
│ Qwen_Qwen3-1.7B-IQ4_XS.gguf    │ 1050 MB│ 充裕 │ 加载│
│ llama-3.2-3b-q4_k_m.gguf       │ 1950 MB│ 超限 │ 加载│ ← 灰色禁用
└──────────────────────────────────────────────────────┘
```

| 适配状态 | 含义 |
|---------|------|
| 🟢 充裕 | 模型在内存预算内 |
| 🟡 临界 | 刚好能放下 |
| 🔴 超限 | 超出预算，禁止加载 |

> 在 **设置 → 内存预算** 中设置可用内存 (MB)，设为 0 则不做限制。

## 5. 设置面板

### 生成参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 最大 Token | 512 | 64–4096 | 每次回复最多生成的 token 数 |
| 上下文长度 | 2048 | 256–32768 | 窗口越大内存消耗越多 |
| 批次大小 | 512 | 64–2048 | 每批处理 token 数 |
| 保留 Token | 4 | 1–100 | 上下文移位时保留的最早 token |

### 采样参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 温度 | 0.7 | 0.0–2.0 | 越高越随机 |
| Top-P | 0.9 | 0.0–1.0 | 核采样阈值 |
| Min-P | 0.1 | 0.0–0.5 | 最低概率阈值 |
| Top-K | 40 | 1–200 | 最高 K 个候选 |
| 重复惩罚 | 1.0 | 1.0–2.0 | 1.0 = 不惩罚 |
| DRY 强度 | 0.0 | 0.0–1.0 | 0.0 = 禁用 |
| XTC 概率 | 0.0 | 0.0–1.0 | 0.0 = 禁用 |

### 模型目录

- 显示当前扫描路径，可直接编辑
- **浏览**：选择文件夹
- **应用并刷新**：重新扫描

### 内存预算

- 设置可用内存上限 (MB)，0 = 不限制
- 模型浏览器根据此值显示适配状态

### 系统提示词

定义模型的角色和行为，点击"应用"生效。

### 模型信息 / 上次推理

- 模型路径、大小、参数量、模板类型
- 上轮推理的 Prompt Token 数、生成 Token 数、速度、首 Token 延迟

## 6. 云端配置

支持连接任何 OpenAI 兼容的 API 服务。

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 接口地址 | API 地址 | `http://localhost:8080/v1/chat/completions` |
| API 密钥 | 密钥（可选） | `sk-xxxx` |
| 模型名称 | 云端模型名 | `gpt-4` |

**工作逻辑**：勾选启用且云端可达 → 请求发云端；不可达 → 自动回退本地；不勾选 → 始终本地。

## 7. 会话管理

### 保存 (Save)

生成两个文件：

| 文件 | 内容 |
|------|------|
| `xxx.json` | 完整对话历史 |
| `xxx.json.ggsn` | KV 缓存二进制快照（恢复后无需重新计算上文） |

### 加载 (Load)

选择 `.json` 文件 → 自动恢复对话和 KV 缓存。如果模型变了，仅恢复文本。

## 8. 多模态数据流输入

### 架构

通过 `MediaSource` 统一抽象管线支持多种输入源：

```
┌──────────────────────────────────────────────────────┐
│                   MediaPipeline                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌───────┐│
│  │ImageSource│  │VideoSource│ │DataStream│  │TextSrc││
│  │  ✅已实现 │  │  🔜预留   │  │  🔜预留  │  │ 🔜预留││
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └──┬────┘│
│       └──────────┬──┴─────────────┴────────────┘      │
│                  ▼                                     │
│         mtmd_tokenize → llama_decode                   │
└──────────────────────────────────────────────────────┘
```

### 输入源状态

| 输入源 | 状态 | 说明 | 实现路线 |
|--------|------|------|---------|
| 🖼️ **ImageSource** | ✅ 已实现 | PNG/JPG/WEBP/BMP | 底层 stb_image.h |
| 🎥 **VideoSource** | 🔜 预留接口 | 视频逐帧分析 | 后续集成 FFmpeg |
| 📷 **CameraSource** | 🔜 预留接口 | 实时摄像头 | 后续集成平台 API |
| 📊 **DataStreamSource** | 🔜 预留接口 | CSV/JSON 数据流注入 | 每批 N 行序列化为文本 |
| 📝 **TextStreamSource** | 🔜 预留接口 | 大文档分块注入 | 每 N token 一块 |

### ImageSource 使用示例

```cpp
#include "media_source.h"

ImageSource img("D:/photo.jpg");
img.open();
MediaFrame frame = img.next_frame();
// frame.data   = RGB 像素数组
// frame.width  = 图片宽度
// frame.height = 图片高度
img.close();
```

### 引擎接入（后续版本）

```cpp
// 编译: cmake .. -DEDGE_GUI_MTMD=ON

EdgeEngine engine;
engine.load_model("qwen2-vl.gguf");
engine.load_mmproj("qwen2-vl.mmproj");   // 加载视觉投影器

ImageSource src("photo.jpg");
src.open();
auto tokens = engine.tokenize_with_media("描述这张图片", src);
// tokens → llama_decode → 生成回复
```

更多细节见 [IMPLEMENTATION_REPORT.md](IMPLEMENTATION_REPORT.md) 中的「多模态数据流输入管线」章节。

## 10. 构建与编译

### 前置条件
- CMake 3.16+
- C++17 编译器 (MSVC 2019+ / GCC 9+ / Clang 10+)

### 编译步骤

```sh
cd llama.cpp
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . --target edge-gui -j 8
```

输出：`build\bin\Release\edge-gui.exe`

### 启用多模态（可选）

```sh
cmake .. -DCMAKE_BUILD_TYPE=Release -DEDGE_GUI_MTMD=ON
cmake --build . --target edge-gui -j 8
```

## 13. 常见问题

**Q: 模型放哪里？**
A: 放 `models\` 目录，启动后点 **模型** 即可看到。也可放任意位置，通过 **打开** 手动选择。

**Q: 模型浏览器显示"超限"且无法加载？**
A: 去 **设置 → 内存预算** 调高可用内存，或设为 0 取消限制。

**Q: 对话太长崩溃了？**
A: KV 缓存会自动移位防止溢出。如果仍然崩溃，在设置中调小"上下文长度"。

**Q: 加载模型时报错？**
A: 确认文件是有效的 GGUF 格式。错误弹窗会显示具体原因。

**Q: 如何启用多模态？**
A: 编译时加 `-DEDGE_GUI_MTMD=ON`，并准备视觉模型 + `.mmproj` 文件。详见上方第 8 章。
